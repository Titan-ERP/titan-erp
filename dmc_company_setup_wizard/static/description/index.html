DMC Company Setup Wizard

Odoo 19 Accounting Administration Multi-Company

A multi-step wizard that automates the full accounting setup for a new company in a multi-company Odoo environment. Eliminates the manual effort of configuring chart of accounts, taxes, journals, and payment providers for each new entity.

Features

  • Creates a new company with address, logo, currency, and optional parent company
  • Auto-generates the next sequential Bank/Cash account code
  • Maps all existing shared Chart of Accounts to the new company (no duplicate records)
  • Copies tax groups and taxes from a selected source company
  • Creates standard journals: Sales, Purchase, Bank, Miscellaneous
  • Creates default payment providers: Wire Transfer, Cash on Delivery, Demo
  • Full rollback on any error — the database is never left in a partial state
  • Accessible via Settings → Technical (debug mode only)

Wizard Steps

Step 1 — Company Information

Enter company name, address, logo, currency, optional parent company, and the source company to copy taxes and payment providers from.

Step 2 — Bank Account Setup

Review or override the auto-generated Bank account code. The code is computed by finding the highest existing Bank/Cash account code and incrementing by 1.

Step 3 — Journal Prefixes

Set unique journal sequence prefixes for Sales, Purchase, Bank, and Miscellaneous journals. Prefixes must be unique across all companies.

Step 4 — Review & Confirm

Review all inputs before committing. A confirmation dialog is shown before any database changes are made.

Step 5 — Done

Success screen showing what was created, with links to the new company and journals.

Configuration

FieldDescription
Company NameRequired. The legal name of the new company.
Copy Setup FromOptional. Source company for copying taxes, tax groups, and payment providers. Leave blank to skip.
Bank Account CodeAuto-generated. Override if the next sequential code is not appropriate.
Parent CompanyOptional. Sets the company hierarchy. When set, Chart of Accounts is inherited automatically — shared account mapping is skipped.
Journal PrefixesMust be unique across all companies. Recommended to append a company abbreviation (e.g. INV-SEL, BNK-SEL).

Multi-Company Behavior

  • Chart of Accounts: Non-Bank/Cash accounts are shared (mapped) to the new company rather than duplicated. Only one new account record is created — the Bank account.
  • Parent Company: When a parent is selected, Odoo automatically inherits the parent's chart. The mapping step is skipped to avoid duplicate code conflicts.
  • Taxes: Copied from the selected source company. Names are made unique per company using an abbreviation suffix when a conflict exists (e.g. 0% (SC)).
  • Payment Providers: Wire Transfer, Cash on Delivery, and Demo are created in Test Mode for the new company.
Debug Mode Required: This wizard is visible only in Odoo debug/developer mode via Settings → Technical → Company Setup Wizard. It is intentionally hidden from regular users to prevent accidental company creation.

Technical Notes

  • All setup steps run inside a single savepoint — any failure rolls back the entire operation.
  • Per-company account code mappings are written atomically with company_ids to satisfy Odoo 19's uniqueness constraint.
  • Tax repartition line account references are intentionally cleared during copy to avoid Odoo's cross-company constraint. These can be set manually after setup.
  • payment.provider.custom_mode is read from the source template at runtime to handle Odoo 19's copy=False behavior on that field.

Dependencies

  • account — Chart of Accounts, Taxes, Journals
  • payment — Payment Providers
  • base_setup — Company configuration
