DMC Backup

Scheduled database backup to Azure Blob Storage or OneDrive

Automatically backs up your Odoo database on a daily schedule. Each backup is compressed, stored in the Odoo filestore, and optionally pushed to Azure Blob Storage or OneDrive (Microsoft Graph API). A built-in history list lets you review past backups, check their size and status, and download them directly from the browser.

Features

  • Daily scheduled backup via Odoo cron
  • Backup history with size, state, and download
  • Configurable retention period (days)
  • Azure Blob Storage destination (SAS token)
  • OneDrive destination (Client Credentials, Microsoft Graph API)
  • Interactive OneDrive folder browser
  • Test Connection button for both destinations

Setup: Azure Blob Storage

1. Create a Storage Account and Container

  1. In the Azure Portal, open Storage accounts and create or select an account.
  2. Inside the account, go to Containers and create a new container (e.g. odoo-backups).
  3. Set the access level to Private.

2. Generate a SAS Token

  1. Open the container, then go to Shared access tokens.
  2. Grant the following permissions: Read, Write, Delete, List.
  3. Set an expiry date far enough in the future (e.g. 2 years).
  4. Click Generate SAS token and URL.
  5. Copy the SAS token (the part starting with sv=..., without the ?).

3. Configure in Odoo

  1. Go to DMC Backup → Configuration and create a new destination.
  2. Set Storage Type to Azure Blob Storage.
  3. Fill in:
    • Storage Account: the account name (e.g. mycompanystorage)
    • Container: the container name (e.g. odoo-backups)
    • SAS Token: the token copied in step 2
  4. Click Test Connection to verify access.
  5. Click Set as Default to use this destination for scheduled backups.

Setup: OneDrive

OneDrive integration uses the Client Credentials (application permissions) OAuth2 flow via Microsoft Graph API. No user sign-in is required — ideal for unattended scheduled backups.

1. Register an Application in Azure Active Directory

  1. In the Azure Portal, open Azure Active Directory → App registrations.
  2. Click New registration.
  3. Give it a name (e.g. Odoo Backup Connection).
  4. Leave the redirect URI blank — it is not needed for this flow.
  5. Click Register.

2. Note the Tenant ID and Client ID

  1. On the app's Overview page, copy:
    • Application (client) ID → this is the Client ID
    • Directory (tenant) ID → this is the Tenant ID

3. Create a Client Secret

  1. Go to Certificates & secrets → Client secrets.
  2. Click New client secret, set a description and expiry.
  3. Copy the Value immediately — it is only shown once.

4. Grant Application Permissions

  1. Go to API permissions → Add a permission → Microsoft Graph → Application permissions.
  2. Search for and add Files.ReadWrite.All.
  3. Back on the API permissions page, click Grant admin consent for [your organisation].
  4. Confirm the status shows a green tick (Granted).
Important: Only Application permissions work for unattended backups. Delegated permissions require an interactive user sign-in and will cause a generalException error at runtime.

5. (Optional) Provision the Target User's OneDrive

If backing up to a specific user's OneDrive and their drive has never been accessed before, ask that user to visit their OneDrive URL once in a browser. Microsoft provisions the drive on first access; the API returns generalException until provisioning is complete.

6. Configure in Odoo

  1. Go to DMC Backup → Configuration and create a new destination.
  2. Set Storage Type to OneDrive.
  3. Fill in:
    • Client ID: from step 2
    • Tenant ID: from step 2
    • Client Secret: from step 3
    • Drive Type: User OneDrive or SharePoint
    • User Email / SharePoint URL: the target email address or full SharePoint site URL
    • Folder Path: optional subfolder (e.g. Backups/Odoo). Use Browse Folders to pick one interactively.
  4. Click Test Connection to verify.
  5. Click Set as Default to activate.
